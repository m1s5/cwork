OPC Historical Data Access Sample Code

February 10, 2007

Overview

This version of the Historical Data Access Sample Code supports .NET 2.0 and has been tested on Windows Vista.

The latest version of this document always on the OPC foundation website.

Any use of the OPC Historical Data Access Sample Code is subject to the terms the OPC Foundation License Agreement.

The OPC Historical Data Access Sample Code consists of an C# client application and a C++ server application. These applications implement the Historical Data Access Custom Interface Specification 1.20. These samples are intended to demonstrate the correct operation of the specification and to serve as starting point for vendors who wish to develop their own implementations.

The sample client is based on the OPC HDA .NET API: a standard .NET class library that provides a simplified, .NET aware interface to the OPC COM servers. The OPC .NET API is available as a separate download on the OPC Foundation website and is maintained as an open source library which is available to all OPC Foundation members.

System Requirements

Operating Systems

All source code provided by the OPC Foundation is only supported on Windows XP SP2 and Windows Vista. That said; the source code has been designed to be portable to other operating systems. For this reason vendors are encouraged to report fixes required to port the samples to other operating systems so they can be incorporated into the baseline and to improve the portability of future sample code.

The samples required the .NET Framework version 2.0. The .NET Framework can be downloaded from here.

All C++/COM applications are compiled as UNICODE applications and will not run on Windows 98 or Windows ME unless they are recompiled. 

Installers

The installers come with a setup.exe file. This is the file that must be used to install under Windows Vista. Double clicking on the MSI will cause an error.

The installer packages are self-installing files; however, they require that Windows Installer 2.0 be present on the system (it is standard on Windows 2000 SP3 and Windows XP). The Windows Installer 2.0 support files for other operating systems can be downloaded from Microsoft's website at:

Usage

Server

The sample server opens and reads the XML file that must be located in the same directory as itself. The XML file must have the same base name as the server (i.e. 'OpcHdaServer'). When executed, the server parses the file a constructs its address space from the items defined within it. 

The server supports two types of items: 

1) Items with static data loaded from a CSV file with same name as the item and in the same directory as the EXE. 

2) Items which generate new data automatically based the value of the 'Derivation Equation' attribute.

The type of a particular item is controlled by the value of the 'Archiving' attribute ('True' indicates that new data is generated).

Client

The sample client can be started from the programs menu. By default, it will connect to the sample server when the ‘connect’ button is clicked. The sample client comes with a standard configuration file that creates several trends. 'Trends' are a concept introduced by the OPC HDA .NET API which combine a set of items together with a set of parameters that can be use to read different types for data from the server. In other words, trends are a type of 'pre-defined query'. 

The user must perform all HDA related tasks by right clicking on items in the tree view on the left side of the application window. Operations may be performed on a server, a trend or an item. 

The set of trends that appear by default are described more fully below. 

Trend

Description

Raw Data

Demonstrates raw data reads with the dataset defined in Section 2.8 of the specification.

Demonstrates processed data reads with the datasets defined in Section 2.9 of the specification.

Demonstrates subscriptions to raw data.

Demonstrates subscriptions to processed data.

Demonstrates playback of raw data.

Demonstrates playback of processed data.

Test Cases

The client application has a menu called 'Test' which executes the test cases defined in the specification and shows the results produced by the server. The menu has three options: one of each sample dataset defined in the specification. The sample code includes a spreadsheet called 'Aggregate Calculations' which contains manual calculations for all the aggregates defined in the specification. Each of the tables in the spreadsheet has a matching test case.

Note that there are some errors in the examples in the specification itself, as a result, the calculations in the spreadsheet are different. In addition, the quality returned by sample server for some aggregates differs from the quality specified in the spreadsheet. These inconsistencies will be resolved once the HDA Working Group has a chance to review the spreadsheet calculations an confirm their accuracy.

Problem Reporting

As part of a broader initiative to increase adoption of OPC specifications and to improve service to members, the OPC Foundation intends to maintain binaries and sample code over time following a model similar to Open Source projects. Vendors are encouraged to report problems, bug fixes, enhancements and any other comments about the components supplied by the OPC Foundation. To support this effort, the OPC Foundation has technical staff prepared to review and incorporate feedback into the baseline and release new versions of the components and sample code in a timely manner for the benefit of all members.

Please note that this support is intended for OPC Foundation members who are incorporating OPC support into their products.
