OPC XML Data Access Sample Code

February 10, 2007

Overview

This version of the XML Data Access 1.00 Sample Code supports .NET 2.0 and has been tested on Windows Vista.

The latest version of this document always on the OPC foundation website.

Any use of the OPC XML Data Access Sample Code is subject to the terms the OPC Foundation License Agreement.

The OPC XML Data Access Sample Code consists of an C# client application, an C# ASP .NET web service application These applications implement the XML Data Access Specification 1.00. These samples are intended to demonstrate the correct operation of the specification and to serve as starting point for vendors who wish to develop their own implementations.

The sample server is written completely with .NET technologies, however, it uses the OPC Data Access 2.05/3.00 Sample Servers as its underlying data source. As a result, this sample demonstrates how vendors can implement XML-DA servers that expose data collected by their existing COM servers.

The sample client is designed to connect to either an XML-DA 1.00 server or a DA 2.XX/3.00 server via a single set of .NET interfaces and classes. As a result, this sample demonstrates how vendors may develop applications that can access core Data Access functionality without having to understand the actual network technology being used. This .NET API will be maintained a separate component that vendors can use as a basis for their own .NET based applications.

Multi-Server Address Space

This version (1.00.3.01) of the sample sample now supports wrapping of multiple COM-DA servers. The address space of each COM-DA server shows up under a branch at the root of the XML-DA server address space. The 'web.config' file for the sample server can be used to control which COM-DA servers are wrapped. 

In addition, the XML-DA server now incorporates a pure .NET implementation of the specification. This implementation uses the .NET API interfaces and can also be accessed by COM-DA clients (see the DA 3.00 sample code for details on the mechanics). 

Complex Data

This version (1.00.2.01) of the sample clients and servers have been modified to support the draft complex data specification. Complex data related features are accessible in the sample client user interface via context menus or buttons used to view or edit an item value. If the item is a complex item (i.e. its defines the complex data properties described in the specification) then a complex item dialog will appear. This dialog displays the complex data in its raw form (binary or text) as well as a tree view. The type description information provided by the server is also accessible via this dialog.

The underlying items exposed by the sample servers are binary values which may be converted to XML by server if requested by the client. The client application is able to display a structured view of the complex data by fetching the OPC Binary type description from the server and using this to parse the buffer returned by the server.

System Requirements

Operating Systems

All source code provided by the OPC Foundation is only supported on Windows XP SP2 and Windows Vista. That said; the source code has been designed to be portable to other operating systems. For this reason vendors are encouraged to report fixes required to port the samples to other operating systems so they can be incorporated into the baseline and to improve the portability of future sample code.

The samples required the .NET Framework version 2.0. The .NET Framework can be downloaded from here.

The XML-DA sample web service requires Internet Information Server (IIS) which is only included with the Windows XP Professional or Windows 2000 operating systems. If IIS is not already installed on a system, it can be installed via the 'Control Panel | Add/Remove Programs' option. Note that the .NET framework must be re-installed after installing IIS on a machine. 

Installers

The installers come with a setup.exe file. This is the file that must be used to install under Windows Vista. Double clicking on the MSI will cause an error.

The installer packages are self-installing files; however, they require that Windows Installer 2.0 be present on the system (it is standard on Windows 2000 SP3 and Windows XP). The Windows Installer 2.0 support files for other operating systems can be downloaded from Microsoft's website at:

Download Windows Installer 2.0 Redistributable for Windows NT 4.0 and 2000

Installation

General

The source code includes a self hosted web service with a dynamically assigned port number. When you run the debugger an ASP .NET tray icon appears. Clicking on this icon will display the base URL that clients need use to connect to the self-hosted server.

The Microsoft SOAP Toolkit 3.0 includes a trace utility that can capture the SOAP requests/responses sent between the client and server. The client can connect to the trace utility by specifying its port number (default is 8080) as part of the server URL.

Proxy Configuration

In most cases, the sample client application will automatically use the Internet Explorer (IE) HTTP proxy configuration. However, some organizations have explicitly prevented web service client applications using the IE settings. As a result, the sample client will not work until the proxy server is specifically entered via the 'Proxy Server..." menu item. This value is saved in the user's application data folder so it only needs to be entered once.

Usage

Server

The sample server is loaded when a client attempts to make a web request. The server is resident with the ASP .NET server process, however, it connects via COM to a local EXE server. The XML-DA sample server uses the same .NET API as the client to communicate with the COM server. The server will remain resident in memory until the ASP .NET framework decides to unload it. All references to the COM server are released at this time.

The actual COM server wrapped by the XML-DA sample server is hard-coded in the web service implementation file. This code can be changed to connect to any DA 3.00 or DA 2.05 compliant server. However, connecting to other COM servers may require changes to the IIS configuration in order to allow the web service to launch the COM server. Further explanation can be found in the Web.config file of the XmlDaSampleServer project.

Client

The sample client can be started from the programs menu. By default, it will connect to the XML-DA sample web service when the ‘connect’ button is clicked. The sample client comes with a standard configuration file that creates four groups on the sample server (described below). The client can connect to any server that supports XML-DA 1.0 or DA 3.0. The "<Browse..>" option in the server combo box allows the client to browse the local network for OPC COM servers on other machines. While navigating the client always try to right click or double click to access functionality (such as viewing the elements in an array value).

The hold time, wait time and ping time are controlled by the subscription update rate and keep alive rate. The update rate determines the hold time. The wait time is difference between the keep alive rate and the update rate. The ping time is the keep alive rate or 5 times the update rate. Items in the update list view are grayed out when a polled refresh completes which contained no update for that item.

Basic Types

This group demonstrates reading and writing of all supported data types, including support for server side data conversions between types as defined in Data Access 2.05 and the writing of quality and timestamp. Right-click on the left tree view to access the read/write functionality. None of the items in this group change, as a result, the server sends a ‘keep alive’ update every 10s (the keep alive rate can be changed in the ‘Edit Group’ dialog).

Deadband

This group demonstrates the deadband specified at the item level. The group has the same item added twice, except one of the items has a deadband of 0% that overrides the group level deadband of 15%. The item changes in sinusoidal pattern so the updates for the item with the group deadband of 15% are not reported when the value is near the peaks of the sinusoid.

Buffering

This group demonstrates the item level sampling rate and buffering. The group has an update rate of 10s and two items that change once every second. The items have buffering enabled, however, one item has a sampling rate of 2s and the other has a sampling rate of 3s. Right click on the right side view and click ‘Keep Old Values’. This displays all values returned in over several updates. The server should deliver an uninterrupted series of 2s samples in blocks of 5 for the item with the 2s sampling rate. The item with the 3s sampling rate has a series of 3s samples in blocks of 5 or 6.

Enumerations

This group demonstrates the use of enumerated items where the set of valid string values is stored in the EU Info property. The four items receive updates from the server both as strings and integer values.

Dynamic Errors

This group demonstrates how dynamic error codes are reported during data change callbacks. It is possible to toggle between display of the localized error texts and the standard error code by right clicking on right pane list. Note, that data change callbacks are only sent if the error state changes.

This group also demonstrates the use of a requested locale (fr-CA) and the server's response with a revised locale (fr). When this group is activated, French error messages should appear in the right pane if the 'Show Error Text' option is enabled via the right pane popup menu.

Complex Data - Binary

This group demonstrates the subscriptions to several complex data items described with the OPC Binary type description system. The 'View' menu option in the popup menu will display dialog that shows the item value in raw an structured format. It also shows the type descriptions provided by the server. These items do not change but it is possible to modify the complex value and write values to the server.

Complex Data - XML

This group demonstrates the subscriptions to several complex data items described with the XML Schema type description system. The 'View' menu option in the popup menu will display dialog that shows the item value as an XML document. It also shows the type descriptions provided by the server. These items do not change. 

The items in this group as represent type conversions for each of the items in the OPC Binary group above. It is possible to enable/disable type conversions via the 'Edit Item' dialog which is accessible when clicking on an item in the tree view on the left.

These items also support Complex Data Filters. These filters may also be entered via the 'Edit Item' dialog. The syntax is any valid XPath that can be applied to the value. For example, the filter '//Uint' when applied to the 'Integers' item will cause the server to only return a value containing elements named 'UInt'.

Complex Data - Dynamic

This group contains one item in binary and XML formats that changes dynamically. The data updates can be monitored in the 'View Complex Data' dialog that is accessible via the 'View' menu in the right hand list view.
